Data Flow — Filter Selection to Render
1 • User

Selects warehouse, category, period, or status
→
2 • State
useFilters()
Updates singleton ref; all views share this state
→
3 • Watch
View Component
Watches getCurrentFilters(); triggers loadData()
→
4 • HTTP
api.js (Axios)
GET /api/endpoint?warehouse=X&category=Y
→
5 • Backend
FastAPI Handler
apply_filters() on in-memory list
→
6 • Validate
Pydantic
Serializes matching items to JSON
→
7 • Render
Template
Computed property re-evaluates; v-for re-renders list

API Endpoints (server/main.py)
Method Path Description Pydantic Model
GET /api/inventory All SKUs (warehouse, category filters) List[InventoryItem]
GET /api/inventory/{id} Single inventory item by ID InventoryItem
GET /api/orders All orders (warehouse, category, status, month) List[Order]
GET /api/orders/{id} Single order by ID Order
GET /api/dashboard/summary KPI aggregates — all 4 filters supported dict
GET /api/demand Demand forecasts (no filters) List[DemandForecast]
GET /api/backlog Backlog items + has_purchase_order flag List[BacklogItem]
GET /api/spending/summary Total procurement, operational, labor, overhead dict
GET /api/spending/monthly Month-by-month cost breakdown List[dict]
GET /api/spending/categories Cost totals per product category List[dict]
GET /api/spending/transactions Recent transaction history List[dict]
GET /api/reports/quarterly Q1–Q4 performance calculated from orders List[dict]
GET /api/reports/monthly-trends Month-by-month order stats List[dict]
POST /api/purchase-orders Create purchase order from backlog item PurchaseOrder
GET /api/purchase-orders/{id} Lookup PO by backlog item ID PurchaseOrder
